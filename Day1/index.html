Task 1: Scan Your Local Network for Open Ports - Report

Objective

The objective of this task was to learn to discover open ports on devices in my local network using Nmap. By performing this network reconnaissance, I aimed to understand the network's exposure and identify potential security risks associated with open ports.

Methodology

  1. Installed Nmap from its official website.
  2. Identified my local IP range using the command ipconfig, which revealed my IPv4 address to be 192.168.1.15. The default gateway was 192.168.1.1. This established the network range as 192.168.1.0/24.
  3. Executed a TCP SYN scan on the network using the command: nmap -sS 192.168.1.0/24.
  4. Saved the results and analyzed the output to identify IP addresses, open ports, and potential vulnerabilities.
  5. Conducted research to understand the services running on the identified ports and the associated security risks.

Analysis of Findings

The scan revealed several active devices on the network. A detailed analysis of the open ports and their associated services is provided below:

IP Address: 192.168.1.1

This IP address belongs to the network's router, which is also the default gateway.

  • Port 23 (Telnet): This is a significant security risk. Telnet sends all data, including login credentials, in plain, unencrypted text, making it vulnerable to eavesdropping and credential theft. This protocol has been largely replaced by the more secure SSH (Secure Shell) protocol. The best practice is to disable the Telnet service and use SSH for remote access if needed.
  • Port 53 (Domain): This port is used for the Domain Name System (DNS) protocol. It is common for a router to act as a local DNS server for network devices. While necessary for network function, an improperly configured DNS server could be exploited for DDoS attacks.
  • Port 80 (HTTP): This port is for Hypertext Transfer Protocol (HTTP), typically used for unencrypted web communication. The router's web-based administration panel likely runs on this port. Using HTTP instead of HTTPS exposes sensitive data (e.g., login credentials for the router) to potential interception.
  • Port 5555 (Freeciv): My research indicates this port is used by the Freeciv game server. This suggests a device on the network is hosting a game. A potential risk is that the game's service could have vulnerabilities that an attacker could exploit to gain unauthorized access to the host device.
  • Port 49152 (unknown): This port falls within the dynamic/private range. These ports are often used for temporary connections and are not officially assigned to a specific service. This could be an RPC (Remote Procedure Call) port used by a Windows system. While the risk is generally lower, the open port could still be a target for attackers if the underlying service is vulnerable.

IP Address: 192.168.1.3

Based on the "iphone-sync" service, this device is likely an Apple iOS device (e.g., iPhone or iPad).

  • Port 62078 (iphone-sync): This port is used by Apple's `usbmuxd` service, which facilitates communication between an iOS device and a host computer (e.g., for iTunes syncing). The service allows for communication and data exchange, and although it is a standard service, it could be a potential point of entry if a vulnerability were to be discovered in the protocol.
  • Port 49152 (unknown): Another dynamic port, likely used by a service on the device for a specific task.

IP Addresses: 192.168.1.5, 192.168.1.7, 192.168.1.13

The scan showed that these devices have all 1000 scanned ports either closed or filtered. This is a good security posture, as it minimizes the attack surface by not exposing unnecessary services to the network.

Conclusion and Recommendations

The network scan was successful in identifying devices and their open ports. The primary security risk identified is the presence of Telnet (port 23) on the router. This unencrypted service poses a significant threat to the network's security.

My recommendations for securing the network include:

  • Disabling the Telnet service on the router immediately and using a more secure alternative like SSH for any remote management needs.
  • Reviewing the router's settings to ensure the administration interface uses HTTPS (Port 443) instead of HTTP (Port 80) to encrypt login credentials.
  • Keeping all network devices and their software updated to patch any known vulnerabilities in services like the Freeciv game or the iOS sync service.
  • Implementing a firewall to block all unnecessary inbound ports and restrict access to services to only trusted devices.
